## Design Context

### Users
Thomas — trail runner y corredor personal en Chile, entrena en los Andes. Usa la app principalmente en desktop para revisar métricas de entrenamiento, preparar carreras y analizar rutas. Contexto de uso: post-entrenamiento o planificación de carrera. Estado mental: enfocado, quiere datos rápidos y accionables.

### Brand Personality
**Limpio, atlético, moderno.** Como Strava pero más personal y sin ruido social. El equivalente visual de un reloj de atleta de gama alta: cada elemento tiene un propósito, los datos son los protagonistas, nada compite con la información.

Tono: directo, preciso, deportivo — no corporativo, no genérico, no juguetón.

### Aesthetic Direction
- **Tema**: Dark mode obligatorio. Fondo profundo ligeramente cálido (teñido hacia el hue naranja/ámbar), no negro puro.
- **Acento**: Naranja/ámbar — `oklch(0.72 0.17 55)`. Usado con precisión (10% regla), no como relleno.
- **Referencia**: Strava — jerarquía visual fuerte, datos destacados, navegación clara, espaciado generoso.
- **Anti-referencia**: Glassmorphism, gradients decorativos, cards apiladas con sombras, cualquier cosa que se vea como "AI UI kit".
- **Tipografía**:
  - Headings y datos grandes: **Barlow Semi Condensed** (weights 400, 600, 700) — atlética, condensa bien sin perder legibilidad.
  - UI y cuerpo: **Manrope** (weights 400, 500, 600) — limpia, excelente rendering de números en dashboards.
  - Ambas desde Google Fonts, sin build step.

### Design Principles
1. **Datos primero**: Los números son los protagonistas. Tipografía grande y bold para métricas clave, sin decoración que compita.
2. **Espaciado con propósito**: Densidad controlada — no abigarrado como Garmin, no vacío. Agrupaciones claras, separaciones que indican jerarquía.
3. **Acento que significa algo**: El naranja/ámbar se usa SOLO para acciones, estados activos y alertas. No como decoración.
4. **Sin ruido visual**: Nada de gradientes decorativos, glassmorphism, sombras llamativas. Los elementos de UI son casi invisibles — dejan que los datos hablen.
5. **Consistencia atlética**: Misma fuente, misma escala, mismo grid en todas las páginas. La coherencia crea confianza.

### Layout Strategy
- Sidebar fija de navegación en desktop (64px icon-only o 220px con labels)
- Contenido principal con padding lateral generoso (px-8 o px-12 en desktop)
- Cards de métricas: sin box-shadow pesado — solo borde sutil (--color-border)
- Grid de 12 columnas con columnas fijas para sidebar + contenido fluido
- Mobile: sidebar colapsa en bottom nav de 5 ítems

### Key Interaction Patterns
- HTMX para sync (botón → spinner → toast de confirmación, sin reload)
- Chart.js con colores del sistema (--color-ctl, --color-atl, etc.)
- Leaflet.js para mapas de actividades
- Tablas con hover sutil (surface-2 background)
- Badges de dificultad GPX: pill con color de fondo según nivel (verde/amarillo/naranja/rojo)
